# AGENTS.md

## Module & Runtime

- Go module: `go_sample_code` (not matching the directory name)
- Go 1.25.0
- No Makefile, Taskfile, or CI — all commands are manual `go` invocations

## Commands

```bash
# Run the server (requires running DB + Redis)
go run cmd/server/main.go -c config/config.local.yaml

# Run all tests
go test ./...

# Run a single package's tests
go test ./internal/middleware/...

# Ent codegen (after editing internal/ent/schema/)
go install entgo.io/ent/cmd/ent@latest
ent generate ./internal/ent/schema

# Format
go fmt ./...
```

## Architecture

Layered design wired with `uber-go/fx` dependency injection:

```
cmd/server/main.go          -- fx app, all providers registered here
internal/handler/{domain}/   -- Fiber HTTP handlers (interface-based)
internal/service/{domain}/   -- Business logic (interface-based)
internal/repo/{domain}/      -- Data access via Ent ORM (interface-based)
internal/middleware/          -- Fiber middleware chain
internal/errno/              -- Typed error codes (HTTP status + business code)
internal/ent/                -- Ent ORM generated code (DO NOT EDIT except schema/)
internal/ent/schema/         -- Ent schema definitions (edit these, then regenerate)
internal/database/           -- DB/Redis client setup and health checks
pkg/                         -- Reusable packages (logger, jwx, trace, metrics, rbac, validator)
config/                      -- YAML config templates
devops/                      -- Docker Compose for local infra (DB, Redis, monitoring)
```

## Key Conventions

- All layer interfaces (Handler, Service, Repo) use constructor injection via `fx.Provide`
- Adding a new domain: create handler/service/repo packages, register in `cmd/server/main.go` fx providers, add routes in `RegisterHooks`
- Error responses use `errno.Decode(data, err)` — returns `{code, message, data}` JSON. New error codes go in `internal/errno/code.go` as package-level vars
- Error codes: 0 = OK, 1000-1099 = common, 1100-1199 = auth, 1200-1299 = file, 1300+ = database
- Middleware order matters: Recovery -> RateLimit -> Metrics -> Logger -> [Auth] -> [RBAC] -> Handler
- Config is YAML with `yaml` struct tags. Config struct lives in `cmd/server/config.go`
- Comments mix Chinese and English — Chinese for inline, English for exported symbols

## Generated Code

`internal/ent/` (except `schema/`) is generated by Ent. Never hand-edit generated files. Only edit:
- `internal/ent/schema/*.go` — entity definitions
- `internal/ent/schema/mixin/` — shared field mixins

No `go:generate` directive exists — run `ent generate` manually after schema changes.

## Local Dev Prerequisites

The server requires PostgreSQL (or MySQL) and Redis running. Quickest path:

```bash
docker compose -f devops/docker-compose.database.yml up -d
```

Full observability stack (Grafana, Prometheus, Tempo, Loki, OTel Collector):

```bash
docker compose -f devops/docker-compose.all.yml up -d
```

Config flag defaults to `config.yaml` in CWD — pass `-c config/config.local.yaml` for local dev.

## Testing

- Uses `testify` for assertions

- No linter config (`.golangci.yml`) — only `go fmt` is expected
