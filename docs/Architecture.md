# 系统架构

## 架构概述

本项目采用分层架构设计，通过依赖注入框架实现各层之间的解耦。

```
┌─────────────────────────────────────────────────────────────┐
│                      Handler Layer                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │   Health    │  │    User     │  │    Future Domains   │ │
│  │   Handler   │  │   Handler   │  │                     │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Service Layer                           │
│  ┌─────────────────────────┐  ┌─────────────────────────┐  │
│  │      User Service       │  │   Future Domain Services │ │
│  └─────────────────────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       Repo Layer                             │
│  ┌─────────────────────────┐  ┌─────────────────────────┐  │
│  │       User Repo         │  │   Future Domain Repos    │ │
│  └─────────────────────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Ent ORM / Database                      │
│  ┌───────────┐  ┌───────────┐  ┌─────────────────────────┐  │
│  │ PostgreSQL│  │   MySQL   │  │        Redis            │ │
│  └───────────┘  └───────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 依赖注入

使用 `uber-go/fx` 实现依赖注入，所有层通过接口抽象。

```
cmd/server/main.go
         │
         ▼
┌──────────────────┐
│   fx.Provide()   │
│   - Logger       │
│   - DB Clients   │
│   - Redis        │
│   - Repos        │
│   - Services     │
│   - Handlers     │
└──────────────────┘
         │
         ▼
┌──────────────────┐
│  fx.Invoke()     │
│  RegisterHooks() │
│  - Route Config  │
│  - Middleware    │
└──────────────────┘
```

## 目录结构

```
go_sample_code/
├── cmd/server/          # 应用入口
│   ├── main.go           # FX 启动 + 路由注册
│   └── config.go        # 配置加载
│
├── internal/             # 内部包（不对外暴露）
│   ├── handler/         # HTTP 处理器层
│   ├── service/         # 业务逻辑层
│   ├── repo/            # 数据访问层
│   ├── middleware/      # 中间件
│   ├── database/       # 数据库客户端
│   ├── ent/           # Ent ORM 代码
│   └── errno/         # 错误码定义
│
├── pkg/                 # 公共包（可被外部引用）
│   ├── logger/         # 日志
│   ├── trace/          # 链路追踪
│   ├── metrics/       # 指标
│   ├── validator/     # 参数校验
│   ├── jwx/           # JWT
│   └── rbac/          # 权限控制
│
├── config/             # 配置文件
└── devops/            # Docker 部署配置
```

## 错误处理

统一的错误码体系：

| 范围 | 类型 |
|------|------|
| 0 | 成功 |
| 1000-1099 | 通用错误 |
| 1100-1199 | 认证授权错误 |
| 1200-1299 | 文件相关错误 |
| 1300-1399 | 数据库错误 |
| 2000+ | 业务错误 |

错误响应格式：

```json
{
  "code": 1303,
  "message": "用户已存在",
  "data": null
}
```
